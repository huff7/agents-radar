    -   **统一化布尔值强制转换** ([PR #99750](https://github.com/openclaw/openclaw/pull/99750)): 解决了核心与插件路径中布尔值强制转换逻辑分散的问题，防止因实现细节不同导致的逻辑漂移。

## 社区热点

今日社区讨论沸点集中在 **Agent 安全** 和 **消息传递控制** 两大主题。

1.  **工具调用文本泄漏（#25592）**:
    -   **链接**: [Issue #25592](https://github.com/openclaw/openclaw/issues/25592)
    -   **热度**: 33 条评论，顶级 `P1` + `diamond lobster` 评级。
    -   **诉求**: 这是社区最焦虑的 UX 问题。当 Agent 在两次工具调用之间产生内部处理文本（如错误信息、日志），这些文本会被错误地投递到用户可见的即时通讯频道（如 Slack）。用户普遍认为这是严重的隐私和安全问题，破坏了与 Agent 交互的沉浸感，并可能导致敏感信息意外泄漏。社区强烈要求引入隔离机制，将内部日志与最终用户输出严格分离。

2.  **输入/输出安全控件（#10659 & #12678）**:
    -   **链接**: [Issue #10659](https://github.com/openclaw/openclaw/issues/10659) | [Issue #12678](https://github.com/openclaw/openclaw/issues/12678)
    -   **热度**: 分别有 13 和 6 条评论，均为 `diamond lobster` 评级。
    -   **诉求**: 社区对 Agent 权限过大的担忧日益增长。`#10659` 要求实现“掩码秘密”系统，让 Agent 能使用 API 密钥但无法读取，防止提示注入攻击。`#12678` 则提出了更广泛的“基于能力的权限模型”，要求对每个 Skill 和 Tool 定义其可以访问的资源（网络、文件、执行命令等），实现默认拒绝高风险操作。

3.  **Agent 消息投递控制（#8299 & #85714）**:
    -   **链接**: [Issue #8299](https://github.com/openclaw/openclaw/issues/8299) | [Issue #85714](https://github.com/openclaw/openclaw/issues/85714)
    -   **热度**: 分别有 7 和 8 条评论。
    -   **诉求**: 用户希望 Agent 对消息的输出有更精细的控制。`#8299` 抱怨子 Agent 每次执行完都会自动发总结消息，非常烦人，希望提供一个配置开关。`#85714` 则是一个更严重的 Bug：当 Agent 忘记调用指定的投递工具时，其最终回复（`agent_message`）会丢失，导致用户收不到回复。这暴露了 Agent 消息投递逻辑的脆弱性。

**分析**：社区热点清晰地指向了**可控性**与**安全性**两大核心。用户不再满足于 Agent“能做什么”，而是迫切希望限制 Agent“不能做什么”，并确保交互过程的透明与可靠。

#### 7. 值得关注的趋势信号

1.  **“可控性”成为核心价值主张**：社区反馈不再是“它听不懂我”，而是“**它又不听我话了**”。从 OpenClaw 的文本泄漏到 Hermes Agent 的权限绕过，再到 CoPaw 的审批绕过，用户对 AI 助手“失控”的恐惧和不满，正在将**安全性**和**权限模型**从一个“加分项”提升为“准入门槛”。未来，能提供成熟、灵活的 RBAC/能力控制框架的项目将获得更强吸引力。

2.  **Runtime 架构迎来“诺曼底时刻”**：多家项目（OpenClaw, NanoBot, CoPaw, ZeroClaw）同时遭遇了**上下文压缩导致“失忆”** 的 Bug，这表明当前主流的“滑动窗口+压缩”运行时策略已接近极限，无法满足长时间、复杂任务的上下文管理需求。下一代的 **“分层记忆”、“自适应上下文”或“显式任务锚点”** 等新范式将成为 AI 智能体性能突破的关键。

3.  **MCP 成为连接生态的“Linux”内核**：尽管 MCP 还很不成熟（多个项目报告了 `conflicting schemas`、工具调用格式错误、插件崩溃），但它已经成为了各项目公认的协议扩展标准。PicoClaw 和 NanoClaw 都在积极支持新的 MCP 传输（HTTP, SSE）；ZeroClaw 则在推进 WASM 插件作为进程外安全执行环境。开源的 MCP 协议是未来“Agent 网络”的事实基建，支持得好坏将决定项目的生态系统繁荣程度。

4.  **模型无关性成为新战场**：NanoClaw 用户报告本地模型因重复发送 Tool Schema 而产生额外成本（`#2917`），IronClaw 用户要求自定义模型协议（`#5609`）。这说明用户不再满足于绑定单一模型，他们希望 Agent 能**智能地、低成本地**切换不同的模型，并自定义后端。项目对模型 Provider 的抽象水平和成本优化能力，将决定其吸引开发者（特别是成本敏感的小团队）的能力。

**结论**：当前 AI 智能体领域正在从 **“能否跑起来”** 向 **“能否安全、可靠、高效地运行起来”** 冲刺。对于开发者而言，选择项目不应仅看功能多少，而应重点关注其**架构的健壮性、对安全问题的响应速度以及 Runtime 的稳定策略**。这将是下一代个人 AI 助手能否从“玩具”走向“工具”的分水岭。
